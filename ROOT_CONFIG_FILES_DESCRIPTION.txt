# Descrição dos Arquivos de Configuração na Raiz

Este documento explica os principais arquivos de configuração localizados no diretório raiz do projeto.

## Arquivos de Configuração:

1. **package.json**:
   - Arquivo de manifesto do Node.js
   - Define dependências, scripts e metadados do projeto
   - Contém comandos para execução, build e desenvolvimento

2. **tsconfig.json**:
   - Configuração do compilador TypeScript
   - Define como o código TypeScript é transpilado para JavaScript
   - Inclui configurações de caminhos, compatibilidade e verificação de tipos

3. **vite.config.ts**:
   - Configuração do Vite (ferramenta de build e desenvolvimento)
   - Define plugins, aliases e otimizações do build
   - Configura o servidor de desenvolvimento

4. **tailwind.config.ts**:
   - Configuração do Tailwind CSS
   - Define temas, cores personalizadas e plugins
   - Configura variações e extensões do framework CSS

5. **postcss.config.js**:
   - Configuração do PostCSS
   - Define plugins de processamento CSS
   - Integra Tailwind CSS com o pipeline de build

6. **eslint.config.js**:
   - Configuração do ESLint
   - Define regras de linting para manter qualidade de código
   - Configura padrões de codificação e boas práticas

7. **components.json**:
   - Configuração do shadcn/ui
   - Define aliases e caminhos dos componentes
   - Configura integração com o projeto

8. **.env**:
   - Variáveis de ambiente
   - Contém chaves de API, URLs e configurações sensíveis
   - Não deve ser versionado (adicionado ao .gitignore)

9. **.gitignore**:
   - Arquivo de ignorados do Git
   - Define quais arquivos/pastas não devem ser versionados
   - Protege informações sensíveis e arquivos temporários

## Arquivos de Documentação:

1. **README.md** - Documentação principal do projeto
2. **TASKS_README.md** - Documentação específica sobre tarefas
3. **SISTEMA_TAREFAS.md** - Documentação do sistema de tarefas
4. **ATUALIZACOES.md** - Registro de atualizações e mudanças
5. **IMPLEMENTATION_PLAN.md** - Plano de implementação
6. **SOLUCAO_ERRO_USEAUTH.md** - Soluções para erros conhecidos

## Propósito:
- Configurar o ambiente de desenvolvimento
- Definir dependências e ferramentas do projeto
- Manter consistência entre diferentes ambientes
- Documentar decisões técnicas importantes

## Características:
- Devem ser mantidos atualizados conforme o projeto evolui
- Alguns arquivos são específicos de ferramentas (Vite, TypeScript, etc.)
- Arquivos de ambiente (.env) devem ser protegidos e não versionados
- Mudanças nestes arquivos podem afetar todo o funcionamento do projeto
