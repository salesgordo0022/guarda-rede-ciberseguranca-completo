# Descrição do Diretório supabase/

Este diretório contém todas as configurações, migrações e scripts relacionados ao Supabase, que é o backend-as-a-service utilizado pela aplicação.

## Estrutura do Diretório:

1. **migrations/** - Scripts de migração do banco de dados
   - Contém arquivos SQL numerados sequencialmente que definem o esquema do banco de dados
   - Cada arquivo representa uma mudança incremental no esquema do banco de dados
   - Os arquivos são executados em ordem numérica para construir o esquema completo

2. **config.toml** - Arquivo de configuração do Supabase
   - Define as configurações do projeto Supabase local
   - Inclui informações de conexão e configurações do serviço

3. **seed.sql** - Script de dados iniciais
   - Contém dados de exemplo ou dados iniciais para popular o banco de dados
   - Usado para configurar um ambiente de desenvolvimento com dados padrão

## Migrações Disponíveis:

1. **20241205_create_tasks_table.sql** - Criação da tabela de tarefas
2. **20241205_system_settings.sql** - Configurações do sistema
3. **20241205_update_activities_table.sql** - Atualização da tabela de atividades
4. **20241205_update_tasks_permissions.sql** - Permissões de acesso às tarefas
5. **20251114145043_66eab815-46d3-43b3-9f61-1eb4eaf870b4.sql** - Migração adicional
6. **20251114145428_40eeccee-f566-435a-8f9e-0b322d26602a.sql** - Outra migração
7. **20251114160000_add_recurrence_fields.sql** - Campos para tarefas recorrentes
8. **20251207_multi_tenant_setup.sql** - Configuração multi-tenant

## Propósito:
- Gerenciar o esquema do banco de dados
- Versionar mudanças no banco de dados
- Facilitar a replicação do ambiente de desenvolvimento
- Manter consistência entre diferentes ambientes (dev, staging, prod)

## Características:
- As migrações são idempotentes e podem ser aplicadas repetidamente
- Seguem o padrão de migrações do Supabase
- Podem incluir criação de tabelas, índices, funções e políticas de segurança
- Suportam tanto migrações para frente quanto para trás (up/down)
